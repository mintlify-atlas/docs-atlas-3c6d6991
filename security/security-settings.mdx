---
title: "Security Settings Reference"
description: "Configure ruby-saml's security parameters for production deployments."
---

ruby-saml exposes two categories of security configuration:

- **`settings.security[...]`** — a Hash of flags that control signing, validation, and encoding behaviour.
- **Top-level settings** — scalar attributes on the `Settings` object that affect parsing and message handling.

All defaults are defined in `OneLogin::RubySaml::Settings::DEFAULTS`.

## `security` hash settings

### Signing flags

<AccordionGroup>
  <Accordion title="security[:authn_requests_signed]">
    **Type:** `Boolean` | **Default:** `false`

    When `true`, ruby-saml signs every `AuthnRequest` sent to the IdP using the
    SP's private key and certificate. The SP's signing certificate is advertised
    in the metadata as `<md:KeyDescriptor use="signing">`.

    ```ruby
    settings.security[:authn_requests_signed] = true
    ```
  </Accordion>

  <Accordion title="security[:logout_requests_signed]">
    **Type:** `Boolean` | **Default:** `false`

    When `true`, ruby-saml signs SP-initiated `LogoutRequest` messages.

    ```ruby
    settings.security[:logout_requests_signed] = true
    ```
  </Accordion>

  <Accordion title="security[:logout_responses_signed]">
    **Type:** `Boolean` | **Default:** `false`

    When `true`, ruby-saml signs `LogoutResponse` messages sent back to the IdP.

    ```ruby
    settings.security[:logout_responses_signed] = true
    ```
  </Accordion>

  <Accordion title="security[:metadata_signed]">
    **Type:** `Boolean` | **Default:** `false`

    When `true`, the SP metadata XML generated by `OneLogin::RubySaml::Metadata`
    includes a `<ds:Signature>` element. Requires `settings.certificate` and
    `settings.private_key` to be set.

    ```ruby
    settings.security[:metadata_signed] = true
    ```
  </Accordion>
</AccordionGroup>

### Assertion requirements

<AccordionGroup>
  <Accordion title="security[:want_assertions_signed]">
    **Type:** `Boolean` | **Default:** `false`

    When `true`, ruby-saml rejects any SAML response whose `<Assertion>` element
    does not carry a `<ds:Signature>`. This adds
    `WantAssertionsSigned="true"` to the SP metadata descriptor.

    ```ruby
    settings.security[:want_assertions_signed] = true
    ```

    If a response passes all other validations but the assertion is unsigned, the
    error is:

    ```
    The Assertion of the Response is not signed and the SP requires it
    ```
  </Accordion>

  <Accordion title="security[:want_assertions_encrypted]">
    **Type:** `Boolean` | **Default:** `false`

    When `true`, ruby-saml rejects any SAML response that does not contain an
    `<EncryptedAssertion>`. This adds `<md:KeyDescriptor use="encryption">` to
    the SP metadata so the IdP knows to encrypt assertions.

    ```ruby
    settings.security[:want_assertions_encrypted] = true
    ```
  </Accordion>

  <Accordion title="security[:want_name_id]">
    **Type:** `Boolean` | **Default:** `false`

    When `true`, ruby-saml rejects any SAML response that does not contain a
    `<NameID>` element in the assertion subject.

    ```ruby
    settings.security[:want_name_id] = true
    ```

    Error when no NameID is found:

    ```
    No NameID element found in the assertion of the Response
    ```
  </Accordion>
</AccordionGroup>

### Certificate expiration checks

<AccordionGroup>
  <Accordion title="security[:check_idp_cert_expiration]">
    **Type:** `Boolean` | **Default:** `false`

    When `true`, ruby-saml raises a `ValidationError` (or appends an error in
    soft mode) if the IdP's X.509 certificate has expired at validation time.

    ```ruby
    settings.security[:check_idp_cert_expiration] = true
    ```

    The error message when the certificate is expired:

    ```
    IdP x509 certificate expired
    ```
  </Accordion>

  <Accordion title="security[:check_sp_cert_expiration]">
    **Type:** `Boolean` | **Default:** `false`

    When `true`, ruby-saml raises `OneLogin::RubySaml::ValidationError` if the
    SP's active certificate has expired. It also filters expired certificates out
    of the metadata and out of the signing/encryption selection logic, so only
    active certificates are used.

    ```ruby
    settings.security[:check_sp_cert_expiration] = true
    ```
  </Accordion>
</AccordionGroup>

### Audience and encoding

<AccordionGroup>
  <Accordion title="security[:strict_audience_validation]">
    **Type:** `Boolean` | **Default:** `false`

    By default, ruby-saml considers an `<AudienceRestriction>` containing only
    empty `<Audience>` elements to be valid. Setting this to `true` makes that
    condition invalid: a non-empty `<Audience>` matching `settings.sp_entity_id`
    is required.

    ```ruby
    settings.security[:strict_audience_validation] = true
    ```

    Error when only empty audiences are present:

    ```
    Invalid Audiences. The <AudienceRestriction> element contained only empty
    <Audience> elements. Expected audience <sp_entity_id>.
    ```
  </Accordion>

  <Accordion title="security[:lowercase_url_encoding]">
    **Type:** `Boolean` | **Default:** `false`

    When `true`, percent-encoded characters in redirect-binding query strings are
    lowercased before signature verification. Required for compatibility with
    ADFS, which URL-encodes SAML data in lowercase.

    ```ruby
    # Required for ADFS compatibility
    settings.security[:lowercase_url_encoding] = true
    ```
  </Accordion>
</AccordionGroup>

### Cryptographic algorithms

<AccordionGroup>
  <Accordion title="security[:digest_method]">
    **Type:** `String` (URI) | **Default:** `XMLSecurity::Document::SHA1`

    The digest algorithm used when signing SP messages and metadata. Accepts any
    of the `XMLSecurity::Document` algorithm URI constants.

    ```ruby
    settings.security[:digest_method] = XMLSecurity::Document::SHA256
    # => "http://www.w3.org/2001/04/xmlenc#sha256"
    ```

    | Constant | URI |
    |---|---|
    | `XMLSecurity::Document::SHA1` (default) | `http://www.w3.org/2000/09/xmldsig#sha1` |
    | `XMLSecurity::Document::SHA256` | `http://www.w3.org/2001/04/xmlenc#sha256` |
    | `XMLSecurity::Document::SHA384` | `http://www.w3.org/2001/04/xmldsig-more#sha384` |
    | `XMLSecurity::Document::SHA512` | `http://www.w3.org/2001/04/xmlenc#sha512` |
  </Accordion>

  <Accordion title="security[:signature_method]">
    **Type:** `String` (URI) | **Default:** `XMLSecurity::Document::RSA_SHA1`

    The signature algorithm used when signing SP messages and metadata.

    ```ruby
    settings.security[:signature_method] = XMLSecurity::Document::RSA_SHA256
    # => "http://www.w3.org/2001/04/xmldsig-more#rsa-sha256"
    ```

    | Constant | URI |
    |---|---|
    | `XMLSecurity::Document::RSA_SHA1` (default) | `http://www.w3.org/2000/09/xmldsig#rsa-sha1` |
    | `XMLSecurity::Document::RSA_SHA256` | `http://www.w3.org/2001/04/xmldsig-more#rsa-sha256` |
    | `XMLSecurity::Document::RSA_SHA384` | `http://www.w3.org/2001/04/xmldsig-more#rsa-sha384` |
    | `XMLSecurity::Document::RSA_SHA512` | `http://www.w3.org/2001/04/xmldsig-more#rsa-sha512` |
  </Accordion>
</AccordionGroup>

## Top-level `soft` setting

**Type:** `Boolean` | **Default:** `true`

The `soft` attribute on `Settings` (not inside the `security` hash) controls how ruby-saml handles signature and certificate validation failures.

- `true` (soft mode, default) — errors are appended to `response.errors`; `is_valid?` returns `false`; no exception is raised.
- `false` (hard mode) — a `OneLogin::RubySaml::ValidationError` is raised on the first validation failure.

```ruby
settings.soft = false  # raise on any validation failure
```

<Note>
  `soft` is a top-level attribute on `Settings`, not a key inside `settings.security`. Set it with `settings.soft = false`, not `settings.security[:soft]`.
</Note>

## Top-level settings

### `message_max_bytesize`

**Type:** `Integer` | **Default:** `250_000`

Maximum number of bytes permitted after decompressing an incoming SAML message.
This limit protects against decompression bomb (zip bomb) denial-of-service
attacks. Increase it only when you have confirmed that legitimate messages from
your IdP exceed the default.

```ruby
settings = OneLogin::RubySaml::Settings.new(message_max_bytesize: 500_000)
```

### `check_malformed_doc`

**Type:** `Boolean` | **Default:** `true`

When `true`, ruby-saml raises an error if Nokogiri reports XML parse errors in
the SAML message. Disabling this allows malformed XML to pass structural
validation, which is not recommended.

```ruby
settings.check_malformed_doc = true  # recommended: keep enabled
```

### `allowed_clock_drift` (Response option)

**Type:** `Numeric` (seconds) | **Default:** `0`

Not a setting on the `Settings` object — passed as an option to the `Response`
constructor. The value is added to the current time when evaluating `NotBefore`
and `NotOnOrAfter` conditions and `SessionNotOnOrAfter`, allowing a small
tolerance for clock skew between SP and IdP servers.

```ruby
response = OneLogin::RubySaml::Response.new(
  params[:SAMLResponse],
  settings:            saml_settings,
  allowed_clock_drift: 1  # tolerate up to 1 second of clock drift
)
```

<Warning>
  Keep `allowed_clock_drift` as small as possible. A large value widens the
  window in which a replayed assertion is accepted. Synchronise server clocks
  with NTP instead of relying on a large drift allowance.
</Warning>

## Recommended production configuration

```ruby
def saml_settings
  settings = OneLogin::RubySaml::Settings.new

  # IdP endpoints
  settings.idp_entity_id       = ENV["IDP_ENTITY_ID"]
  settings.idp_sso_service_url = ENV["IDP_SSO_URL"]
  settings.idp_slo_service_url = ENV["IDP_SLO_URL"]

  # Full certificate — never use fingerprint in production
  settings.idp_cert = ENV["IDP_CERT"]

  # SP identity
  settings.sp_entity_id                  = "https://yourapp.example.com/saml/metadata"
  settings.assertion_consumer_service_url = "https://yourapp.example.com/saml/consume"
  settings.single_logout_service_url      = "https://yourapp.example.com/saml/logout"

  # SP signing and decryption credentials
  settings.certificate = ENV["SP_CERT"]
  settings.private_key = ENV["SP_PRIVATE_KEY"]

  # Security flags
  settings.security[:authn_requests_signed]   = true
  settings.security[:logout_requests_signed]  = true
  settings.security[:logout_responses_signed] = true
  settings.security[:want_assertions_signed]  = true
  settings.security[:want_assertions_encrypted] = true
  settings.security[:metadata_signed]         = true
  settings.security[:check_idp_cert_expiration] = true
  settings.security[:check_sp_cert_expiration]  = true
  settings.security[:strict_audience_validation] = true

  # Use SHA256 instead of the SHA1 default
  settings.security[:digest_method]    = XMLSecurity::Document::SHA256
  settings.security[:signature_method] = XMLSecurity::Document::RSA_SHA256

  # Hard mode: raise on any validation failure
  settings.soft = false

  settings
end
```

<Note>
  Load certificates and private keys from environment variables or a secrets
  manager rather than committing them to source control.
</Note>
