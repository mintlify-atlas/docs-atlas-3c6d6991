## 1.18.1 — Jul 29, 2025

<Warning>
  **CVE-2025-54572** — This release fixes a denial-of-service vulnerability caused
  by large SAML messages. Upgrade immediately if you are on any 1.18.x release.
</Warning>

- Fix vulnerability CVE-2025-54572: prevent DOS due to excessively large SAML messages.
- Adapt tests to execute signature validation earlier in the pipeline.
- CI improvements and Ruby 3.4 support added.

---

## 1.18.0 — Mar 12, 2025

<Warning>
  **CVE-2025-25291 / CVE-2025-25292** — SAML authentication bypass via Signature
  Wrapping attack caused by a parser differential. **CVE-2025-25293** — potential
  denial-of-service by abusing compressed SAML messages. Upgrade from any earlier
  version is strongly recommended.
</Warning>

- Fix CVE-2025-25291, CVE-2025-25292: SAML authentication bypass via Signature Wrapping attack.
- Fix CVE-2025-25293: potential DOS via abuse of compressed SAML messages.
- Add support for retrieving `AuthnInstant` and `AuthnContextClassRef` values from `SAMLResponse`.
- Fix ambiguous regex warnings.
- Fix typo in `SPNameQualifier` error text.

See [Upgrading 1.17.x → 1.18.0](/migration/upgrading) for migration details.

## 1.17.0 — Sep 10, 2024

<Warning>
  **CVE-2024-45409** — Critical vulnerability: SAML authentication bypass via
  incorrect XPath selector. All users on earlier versions should upgrade.
</Warning>

- Fix critical vulnerability CVE-2024-45409: SAML authentication bypass via incorrect XPath selector.
- Add `Settings#sp_cert_multi` parameter to facilitate SP certificate and key rotation.
- Support multiple simultaneous SP decryption keys via `Settings#sp_cert_multi`.
- Deprecate `Settings#certificate_new` parameter in favor of `Settings#sp_cert_multi`.
- `:check_sp_cert_expiration` now uses the first non-expired certificate/key when signing or decrypting; raises an error only when no valid certificates remain.
- `:check_sp_cert_expiration` now also validates the certificate `not_before` condition (previously only `not_after` was checked).
- `:check_sp_cert_expiration` causes the generated SP metadata to exclude inactive or expired certificates.
- Add CI coverage for Ruby 3.3 and Windows.

## 1.16.0 — Oct 09, 2023

## 1.15.0 — Jan 04, 2023

- Replace `strip!` with `strip` in the `compute_digest` method to avoid frozen string errors.
- Fix `dateTime` format for the `validUntil` attribute in generated SP metadata.
- Support `Settings#idp_cert_multi` with string keys (in addition to symbol keys).
- Improve code quality throughout the library.
- Update repository and maintainer information; add Coveralls support.
- Fix tests and dependency adjustments; add Ruby 3.2 and new JRuby versions to CI.

---

## 1.14.0 — Feb 01, 2022

- Support escape downcasing when validating SLO signatures from ADFS/Azure.
- Support the ability to change the SAML message ID prefix.
- Make the UUID editable on SAML messages generated by the toolkit.
- Add security setting to more strictly enforce audience validation.

---

## 1.13.0 — Sep 06, 2021

- Replace the `MAX_BYTE_SIZE` constant with a configurable setting: `message_max_bytesize`.
- `:allowed_clock_drift` is now bidirectional.
- Support `:name_id_format` option for `IdpMetadataParser`.
- `IdpMetadataParser` now always sets `idp_cert_multi`, even when there is only one certificate.
- Add new IdP SSO/SLO binding params (`idp_sso_service_binding`, `idp_slo_service_binding`), deprecating `:embed_sign`.
- Refactor the `OneLogin::RubySaml::Metadata` class.
- Support milliseconds in `cacheDuration` parsing.
- Do not append ` | ` to `StatusCode` unnecessarily.
- Add warning about `IdpMetadataParser` and SSRF risk.
- CI: migrate from Travis CI to GitHub Actions.

## 1.5.0 — Aug 31, 2017
