Ruby SAML gives you the tools to add SAML-based Single Sign-On (SSO) to any Ruby or Rails application. It handles the client side of SAML 2.0 — generating authentication requests, validating IdP responses, managing Single Logout, parsing IdP metadata, and securing message exchange with XML signatures and encryption.

## What ruby-saml does

Ruby SAML implements the **service provider (SP) side** of SAML 2.0. Your application acts as the SP, and your users authenticate through an external **Identity Provider (IdP)** such as Okta, Azure AD, OneLogin, or any other SAML-compliant IdP.

The library manages:

- **SSO initialization** — Building and signing `AuthnRequest` messages to redirect users to the IdP
- **Response validation** — Parsing, decrypting, and cryptographically verifying IdP `SAMLResponse` messages
- **Single Logout (SLO)** — SP-initiated and IdP-initiated logout flows
- **IdP metadata parsing** — Automatically loading IdP configuration from metadata XML or remote URLs
- **SP metadata generation** — Exposing your SP configuration as a metadata XML endpoint
- **Signing and encryption** — Signing SP requests and decrypting `EncryptedAssertion` elements

## Security

Ruby SAML has a track record of active security maintenance. Always run the latest release. Critical CVEs are patched promptly — see the [Changelog](/migration/changelog) for details.

<Warning>
  Versions before 1.18.0 contain critical authentication bypass vulnerabilities (CVE-2025-25291, CVE-2025-25292, CVE-2025-25293). Upgrade to **1.18.1** or later immediately.
</Warning>
